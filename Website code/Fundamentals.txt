            <h2>What Are Progressive Web Apps?</h2>
            <p>
                Progressive Web Apps (PWAs) are web applications that use modern web capabilities to deliver 
                app-like experiences to users. They are built using standard web technologies including HTML, 
                CSS, and JavaScript, but behave like native applications on mobile and desktop platforms.
            </p>
            <p>
                The term "progressive" refers to the fact that these apps work for every user, regardless of 
                browser choice, because they are built with progressive enhancement as a core principle. They 
                progressively add features based on the capabilities of the user's device and browser.
            </p>
        </section>

        <!-- Core characteristics -->
        <section>
            <h2>Core Characteristics of PWAs</h2>
            <ul class="content-list">
                <li><strong>Progressive:</strong> Work for every user, regardless of browser choice</li>
                <li><strong>Responsive:</strong> Fit any form factor - desktop, mobile, tablet</li>
                <li><strong>Connectivity Independent:</strong> Work offline or on low-quality networks</li>
                <li><strong>App-like:</strong> Feel like an app with app-style interactions and navigation</li>
                <li><strong>Fresh:</strong> Always up-to-date thanks to service worker update process</li>
                <li><strong>Safe:</strong> Served via HTTPS to prevent tampering</li>
                <li><strong>Discoverable:</strong> Identifiable as applications via W3C manifests</li>
                <li><strong>Re-engageable:</strong> Make re-engagement easy through features like push notifications</li>
                <li><strong>Installable:</strong> Can be installed on device home screen without app store</li>
                <li><strong>Linkable:</strong> Easily shared via URL without complex installation</li>
            </ul>
        </section>

        <!-- Building blocks -->
        <section>
            <h2>Essential Building Blocks</h2>
            
            <h3>1. Service Workers</h3>
            <p>
                Service workers are JavaScript files that run separately from the main browser thread, intercepting 
                network requests, caching resources, and enabling background sync. They are the backbone of PWA 
                offline functionality and provide capabilities that were previously exclusive to native apps.
            </p>

            <h3>2. Web App Manifest</h3>
            <p>
                The web app manifest is a JSON file that tells the browser about your web application and how it 
                should behave when installed on the user's device. It controls how the app appears to users, 
                defines its start URL, display mode, icons, and theme colors.
            </p>

            <h3>3. HTTPS</h3>
            <p>
                PWAs must be served over HTTPS to ensure security and privacy. This requirement protects users from 
                man-in-the-middle attacks and ensures that service workers can only be registered on secure origins.
            </p>
        </section>
